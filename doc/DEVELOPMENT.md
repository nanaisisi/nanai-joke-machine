# NanAI Joke Machine 開発計画

## 概要

NanAI Joke Machine の開発ロードマップと将来の拡張計画。

## 現在の安定版（v0.1.0）

### ファイル構造

- `src/main.rs` - メインジョークプログラム
- `config.toml` - チャネル設定
- `profiles/` - プロファイル設定ファイル

### 特徴

- チャネルベースのジョークメッセージ出力
- プロファイルごとのハードウェアシミュレーション
- 非同期タスクによる継続実行

### 課題

- メッセージのカスタマイズ不足
- GUI インターフェースの欠如
- エラーハンドリングの改善
- 拡張チャネルの実装

## 次世代バージョン（v1.0.0）

### 目標

拡張性とユーザビリティを向上させたジョークプログラムの実装。

### 主要機能

- **ch7 拡張**: ログ集約やユーザーインタラクションの実装
- **GUI インターフェース**: TUI または Web UI の追加
- **設定拡張**: チャネルごとの出力頻度設定
- **新チャネル**: AI アシスタントやネットワークスキャンなどの追加
- **プロファイル管理**: 動的プロファイル切り替え

## 開発ブランチ運用

### 推奨ワークフロー

1. `main` ブランチは常に安定版を維持
2. 新機能開発は `feature/` プレフィックスのブランチで実施
3. 破壊的変更の場合は `breaking-change/` ブランチを使用
4. リリース準備ができたら `develop` ブランチを経由して `main` にマージ

### 現在の開発ブランチ

- `feature/config-based-profiles` - プロファイル設定ファイル化

## 注意事項

- ジョークプログラムの性質を維持し、本物のセキュリティ機能を実装しない
- ユーザーのプライバシーを尊重
- コードの簡潔さと分割を重視
