# NanAI Joke Machine

NanAI Joke Machine は、ジョークプログラムとして設計された Rust アプリケーションです。セキュリティシステムを模倣したチャネルベースのメッセージ出力を特徴とします。

## 機能概要

このプログラムは、以下のチャネルを備えています：

- **ch0**: メインシステム報告（損傷報告、基本的に使用しない）
- **ch1**: パッシブセキュリティ報告（定時報告、侵入阻止など）
- **ch2**: 使用率報告（負荷率の定時報告）
- **ch3**: アクティブセキュリティ報告（敵システムの捜索、破壊など）
- **ch4**: ネットワーク内端末の管理、保護、検閲
- **ch5**: ファイルサーバー（圧縮など）
- **ch6**: バイナリリポジトリ（プログラムの保存と配布）
- **ch7**: 未定義（将来拡張用）

各チャネルは ON/OFF 可能で、開始時にアクティブに合成されます。開始メッセージの ON/OFF も設定可能です。

このプログラムは、いわゆる「仕事をしていることを偽装するためのツール」を参考に設計されたジョークプログラムです。実際のセキュリティ機能は持たず、エンターテイメント目的のみです。

## プロファイル

複数のハードウェアプロファイルをサポート。各プロファイルは `profiles/` ディレクトリ内の個別 TOML ファイルとして保存されます。

- **default**: 将来型重一般コンピューター
- **test2**: 将来型軽一般コンピューター
- **test3**: 現用型業務サーバーコンピューター
- **test4**: 現用型軽量 PC

## ビルドと実行

### 依存関係

- Rust 1.70+
- tokio, serde, toml, rand

### ビルド

```bash
cargo build --release
```

### 実行

デフォルトプロファイルで実行：

```bash
cargo run
```

特定のプロファイルで実行：

```bash
cargo run <profile_name>
```

例：

```bash
cargo run test2
```

## 設定

各プロファイルは `settings/` ディレクトリ内の個別 TOML ファイルとして保存されます。

### プロファイル設定例（settings/default.toml）

```toml
cpu_cores = 16
cpu_breakdown = "4:8:4"
memory_gb = 128
npu_count = 2
npu_breakdown = "1:1"
npu_tops = 50
gpu_count = 2
gpu_breakdown = "1:1"
gpu_tops = 262
network_mbps = 19600
wireless = true
```

## 動作

プログラムは非同期タスクとして各チャネルを実行し、ランダムなタイミングでジョークメッセージを出力します。実際のセキュリティ機能は持たず、エンターテイメント目的です。

## 拡張提案

- ch7 の機能追加（例: ログ分析チャネル）
- GUI インターフェースの追加
- 実際のシステム監視機能の統合（オプション）
- ネットワーク機能の拡張（ch4, ch5, ch6）

## ライセンス

このプロジェクトはジョークプログラムとして公開されています。いわゆる「仕事をしていることを偽装するためのツール」を参考に設計されており、実際のセキュリティシステムとして使用しないでください。
